# Obsidian brain — DogShift founder vault

## Critical context

`brain/` is the founder's (Alexis) personal Obsidian vault. **Gitignored**. NOT source of truth — first-person notes, sitter fiches, daily journal, ideas, decisions.

You can read it for context (*"there's a sitter Sonia who reported X"*) but **never assume its content reflects production state** — it's notes, not authoritative DB content.

## Folder structure

```
brain/
├── 🤖 Agents/          — agent fiches + Telegram bots + Skills
├── 🐛 Bugs/             — bug workflow + lessons learned
├── 👥 Pilote/           — sitter fiches (one per active sitter)
│   └── Tous les sitters actifs/
├── 💡 Idées/            — brainstorming, ideas, RFC drafts
├── 🗺️ Roadmap/          — planning, milestones
├── 🧠 Décisions/        — ADRs (architecture decision records)
├── 📓 Journal/          — daily notes (date-keyed)
├── 📥 Inbox/            — unsorted (target empty)
├── ⚙️ Templates/        — Obsidian templates (daily, sitter, idea, decision)
└── DogShift Brain.md    — entry note, graph hub
```

## Color-coding (graph view)

Defined in `.obsidian/graph.json` :
- 🟠 Bugs (`brain/🐛 Bugs`)
- 🔵 Pilote / sitters (`brain/👥 Pilote`)
- 🟣 Décisions (`brain/🧠 Décisions`)
- 🟡 Idées (`brain/💡 Idées`)
- 🟢 Journal (`brain/📓 Journal`)
- 🩷 Agents (`brain/🤖 Agents`)
- ⚫ docs (`docs`)
- ⚪ Inbox / Templates (gray)

When adding files, place them in the right folder so they get the color. Uncolored = problematic structure.

## Available slash-commands (operate on brain/ exclusively)

| Command | Use when |
|---|---|
| `/ingest <source>` | User pastes raw content (URL, text, conversation log) → structure it into Obsidian notes |
| `/lint` | Audit brain/ structure (orphans, dead links, format inconsistencies) |
| `/query <question>` | Search across notes to answer a question |
| `/save <decision>` | Capture a decision in `brain/🧠 Décisions/` with proper ADR format |

These slash-commands NEVER touch code. They operate on `brain/` and `.obsidian/`.

## Linking convention

- Wikilinks `[[Note title]]` everywhere — Obsidian convention
- Every note ends with a `## Liens` section listing related notes
- Use `[[Folder/Note]]` only if name is ambiguous, otherwise just `[[Note]]`

Example footer :
```markdown
## Liens

- [[Telegram bots]] · [[Format Telegram]]
- [[Agents]]
- [[Bot maintenance]]
```

## Sitter fiches (`brain/👥 Pilote/Tous les sitters actifs/`)

Auto-generated by a sync script from prod DB. Format :

```markdown
---
auto_synced: true
synced_at: 2026-05-18
sitter_id: s-1774429173366-a78495f20b5a2
---

# Lucie Lausanne

> Fiche générée depuis la prod DB. Pour la garder, supprime la ligne `auto_synced: true` du frontmatter et ajoute tes notes en dessous.

## Coordonnées
…
```

If `auto_synced: true`, the file may be overwritten on next sync. To preserve manual edits, remove that line.

## Bug fiches (`brain/🐛 Bugs/`) vs `docs/bugs/`

Two parallel locations :

- **`docs/bugs/`** : in-repo, committed. Each fiche has `## 🤖 Automated detection` block. Read by nightly cron `/api/cron/bug-regression-check`. **Source of truth for the playbook.**
- **`brain/🐛 Bugs/`** : Obsidian, gitignored. Personal notes about the debugging journey, what tripped up the founder. Optional but useful for "the human side" of an incident.

When fixing a bug, ALWAYS create the `docs/bugs/` fiche. Brain fiche is bonus.

## Agent fiches (`brain/🤖 Agents/`)

One per autonomous agent in DogShift. Pattern :
- Schedule / trigger
- What it does (3-5 bullets)
- Side effects (emails, Telegram, DB writes)
- Failure mode + recovery
- Links to related agents + slash commands

Sub-folders : `Telegram bots/` (6 bots), `Skills/` (Claude skills).

## What NOT to do

- ❌ Edit `brain/` from a code-review context — that's the founder's personal space
- ❌ Treat `brain/` content as authoritative for prod state — always verify against DB/Sentry
- ❌ Commit `brain/` to git — `.gitignore` excludes it. Check before any `git add`.
- ❌ Restructure folders without asking — Alexis has a workflow built around the emoji folders
- ❌ Use the slash-commands on non-brain content (they're scoped to brain/ only)

## Audit checklist (when running `/lint`)

- Files in 📥 Inbox older than 7 days → either move or delete
- Orphan files in 🤖 Agents/ root that aren't subfolders/skills
- Duplicate sitter fiches (one in 👥 Pilote root + one in Tous les sitters actifs/)
- Notes without `## Liens` footer
- Empty notes (0 bytes, leftover from Obsidian "New note")
- Files outside the color-coded folders (uncolored in graph view)

The May 19 2026 cleanup removed 5 such orphans/duplicates. Run periodically.
