# Native app — Capacitor playbook

DogShift native iOS / Android apps wrap the existing Next.js web app via **Capacitor** in **remote-URL mode** : the WebView loads `https://www.dogshift.ch` directly, so SSR + 218 API routes + Auth.js stay intact. Native plugins layered on top (push notifications, deep links, status bar, splash) provide the "real app" experience required by Apple's App Store review.

## Why remote-URL mode (not static export)

`output: "export"` would strip Auth.js, all `/api/*` routes, and ~26 SSR pages. Non-starter for DogShift. The web app stays a Next.js / Vercel deploy ; the native shells just open it.

Trade-off : the app is dead without internet (same as any browser). We mitigate with a splash screen and the native push notifications.

## File map (in this repo)

| File | Purpose |
|---|---|
| `capacitor.config.ts` | Capacitor settings (appId, remote URL, plugins, theme) |
| `capacitor-web-build/index.html` | Pre-redirect fallback served if the app boots offline |
| `ios/` | Xcode project (generated by `npx cap add ios` after CocoaPods install) |
| `android/` | Android Studio project (generated by `npx cap add android`) |
| `public/.well-known/apple-app-site-association` | Universal Links manifest |
| `public/.well-known/assetlinks.json` | Android App Links manifest |
| `lib/native/capacitorBridge.ts` | Client-side init (push register, deep links, status bar) |
| `lib/push/native.ts` | Server-side APNs + FCM senders |
| `lib/push/unified.ts` | Wrapper that fans out to web push + APNs + FCM |
| `app/api/native/register-push-token/route.ts` | Backend endpoint the app calls after APNs/FCM token receipt |
| `prisma/schema.prisma → NativePushToken` | Token storage |
| `scripts/generate-app-icons.sh` | Generates iOS + Android + PWA icons from `assets/app-icon-master.png` |

## Setup checklist (one-time)

### 1. Install local tooling

```bash
# Required by Capacitor iOS
sudo gem install cocoapods

# JDK for Android Gradle
brew install openjdk@17
# Then add to ~/.zshrc : export PATH="/opt/homebrew/opt/openjdk@17/bin:$PATH"
```

Xcode + Android Studio install themselves via Mac App Store / [developer.android.com/studio](https://developer.android.com/studio).

### 2. Generate the iOS project

```bash
npx cap add ios
```

This creates the `ios/` folder. Commit it.

### 3. Drop the master app icon

Drag your 1024×1024 PNG (NO transparency for Apple) into :

```
assets/app-icon-master.png
```

Then run :

```bash
./scripts/generate-app-icons.sh
```

This generates all required sizes for iOS (`ios/App/App/Assets.xcassets/AppIcon.appiconset`), Android (`android/app/src/main/res/mipmap-*`), and PWA (`public/pwa-icons/`).

### 4. Configure Apple Developer account ($99/year)

1. Create an App ID `ch.dogshift.app` at [developer.apple.com/account/resources/identifiers](https://developer.apple.com/account/resources/identifiers/list)
2. Enable capabilities : **Push Notifications**, **Sign in with Apple**, **Associated Domains** (for Universal Links).
3. Create an **APNs Auth Key** (Keys → +) — download the `.p8` file (you only get it once).
4. Note the **Key ID** (10 chars) and your **Team ID** (10 chars).

### 5. Configure Sign in with Apple

1. Create a **Service ID** `ch.dogshift.app.signin` (separate from the App ID).
2. Enable **Sign In with Apple** on the Service ID with primary App ID `ch.dogshift.app`.
3. Add return URL : `https://www.dogshift.ch/api/auth/callback/apple`.
4. Generate the **client secret JWT** with your private key — it's valid 6 months max. Auth.js docs : <https://authjs.dev/getting-started/providers/apple>
5. Set `AUTH_APPLE_ID` (= Service ID) + `AUTH_APPLE_SECRET` (= the JWT) on Vercel.

### 6. Configure Firebase / FCM ($0)

1. Create a Firebase project at [console.firebase.google.com](https://console.firebase.google.com).
2. Add an Android app with package `ch.dogshift.app`.
3. Download `google-services.json` → drop into `android/app/`.
4. Project settings → Service accounts → Generate new private key → JSON file.
5. `base64 -i firebase-service-account.json` → set as `FCM_SERVICE_ACCOUNT` env on Vercel.
6. Set `FCM_PROJECT_ID` to your Firebase project id.

### 7. Configure APNs (env vars)

```bash
APNS_KEY_ID=ABCDEF1234            # 10-char Apple Key ID
APNS_TEAM_ID=XYZ987WVU6           # 10-char Apple Team ID
APNS_KEY_P8=<base64 of .p8 file>  # base64 -i AuthKey_XXX.p8
APNS_BUNDLE_ID=ch.dogshift.app
APNS_USE_SANDBOX=1                # "1" for TestFlight, "" for App Store
```

### 8. Replace placeholders in Universal Links

`public/.well-known/apple-app-site-association` — replace `REPLACE_WITH_TEAM_ID` with your Apple Team ID (e.g. `XYZ987WVU6.ch.dogshift.app`).

`public/.well-known/assetlinks.json` — get the SHA-256 fingerprint from Google Play Console → Setup → App signing → SHA-256 certificate fingerprint, paste into `sha256_cert_fingerprints`.

### 9. Sync + open in IDE

```bash
npx cap sync
npx cap open ios       # opens Xcode
npx cap open android   # opens Android Studio
```

In Xcode :
- Project → Signing & Capabilities → set your Team.
- Add capabilities : **Push Notifications**, **Sign in with Apple**, **Associated Domains** (add `applinks:www.dogshift.ch` and `applinks:dogshift.ch`).

## Daily workflow

```bash
# Web work proceeds as usual — Vercel handles everything.
# After ANY change to capacitor.config.ts or native plugins :
npx cap sync

# Test locally on simulator/emulator :
npx cap run ios
npx cap run android

# Test on a real device :
# - iOS : Xcode → Play button with device selected
# - Android : Android Studio → Run with device connected (USB debugging enabled)
```

## Publishing

### iOS — TestFlight first, then App Store

1. Xcode → Product → Archive → Distribute App → App Store Connect → Upload.
2. App Store Connect → TestFlight → add internal testers (you).
3. Test for 2 weeks minimum.
4. App Store Connect → Submit for Review (provide demo account credentials, privacy URL = `https://www.dogshift.ch/confidentialite`, support URL = `https://www.dogshift.ch/contact`).

### Android — Internal testing → Production

1. Android Studio → Build → Generate Signed Bundle → AAB (Android App Bundle).
2. Play Console → Production → Create new release → Upload AAB.
3. Start with internal testing track for ~1 week.
4. Promote to Production track.

## Apple App Review tripwires (anticipate)

| Issue | Resolution |
|---|---|
| **"App is a website wrapper, rejected per 4.2"** | Native features we ship : push notifications, Sign in with Apple, Universal Links, native splash, status bar theming. Should be enough. If not, add biometric login. |
| **"Account deletion missing"** | We ship `/account/settings` with the deletion flow that calls `DELETE /api/account/delete`. Apple-compliant since 2022. |
| **"Privacy Policy missing"** | `/confidentialite` URL provided in App Store Connect listing. |
| **"In-app purchase violation (Stripe)"** | DogShift sells real-world services (dog-sitting), not digital goods. Out of Apple's IAP scope. **Critical** : the booking checkout must open in Safari external (use `openExternalBrowser()` from `lib/native/capacitorBridge.ts`), NOT in the in-app WebView. |
| **"Sign in with Apple required"** | Implemented in `auth.ts`. Falls back to no-op if env vars missing (so the web stays usable). |
| **"Push notif opt-in pre-permission UX"** | We trigger `PushNotifications.requestPermissions()` only after the user has explored a bit (no-op currently — see TODO below). |

## Stripe checkout — Apple compliance

When the user taps a "Réserver" button inside the native app, **the checkout MUST open in the system Safari, not the in-app WebView**. Otherwise Apple flags it as in-app purchase of a digital good and demands 30%.

Implementation : the `openExternalBrowser()` helper in `lib/native/capacitorBridge.ts` uses `@capacitor/browser` to open in SFSafariViewController. To use it, refactor the booking flow's CTA to detect native mode and call this helper instead of a direct `<a>` link.

## Push notifications — sending from cron / agents

Existing callers using `sendPushToUser()` (web-only) keep working unchanged. To also send to native devices :

```ts
import { sendPushToUserAllChannels } from "@/lib/push/unified";

await sendPushToUserAllChannels(userId, {
  title: "Nouvelle réservation",
  body: "Marie a réservé pour vendredi 18h",
  url: "/host/requests/abc123",
  tag: "request-abc123",
});
```

Returns `{ web: { sent, failed }, ios, android, nativeFailed }`. Cron jobs should `await` it and persist counts to `AgentLog`.
